A talk given by Xinyuan Guo about their third year project.

# Abstract
Optimisation algorithms are often accompanied by convergence proofs that are mathematically standard but long and difficult to reuse in machine-checked form. This project investigates how selected optimisation guarantees can be reconstructed and organised in Lean 4 on top of an existing upstream formalisation ecosystem. Rather than treating prior results as black-box final theorems, the project rebuilds selected proof chains from earlier derivational cut points while reusing existing algorithm definitions and analytical infrastructure where appropriate. On this basis, the project extracts reusable proof components for recurring steps such as step-size rewriting, telescoping sums, output-rule conversion, weighted-potential arguments, proximal reasoning, and algebraic normalisation. These components are then exposed through a lightweight framework that separates algorithm traces, assumption packs, certified property statements, and registry-based discovery. The resulting prototype supports certified properties across gradient descent, subgradient, and accelerated Nesterov-style families, and demonstrates both menu-based and direct certification routes. The main contribution of the project is not a new optimisation theorem, but a reusable proof-engineering organisation for machine-checked optimisation guarantees in Lean 4. More specifically, the dissertation contributes (1) derivation-level reproductions of selected convergence proofs, (2) reusable abstractions validated across multiple proof lines, and (3) a sparse certified property space with registry-aware discovery and certification. Limitations include the current reliance on selected upstream infrastructure, the absence of advanced meta-programmed automation, and a registry that represents implemented properties rather than a full pairwise comparison engine.
